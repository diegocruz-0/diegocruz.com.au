FSLogix in Hybrid Identity AVD: What Nobody Tells You

After deploying AVD in a hybrid identity environment, I ran into three FSLogix issues that cost me two days. Here's what I learned — so it costs you nothing.

azure avd fslogix identity

The Setup

Standard hybrid identity AVD deployment: pooled host pool, Windows 11 multi-session, FSLogix profile containers on Azure Files, GPO applied via on-prem AD. Everything works in the lab. Then you hit production.

// environment
AVD pooled host pool · Windows 11 multi-session · Hybrid join · Azure Files (Kerberos auth) · FSLogix 2201 · Azure AD Connect sync

Problem 1 — The Silent Mount Failure

FSLogix mounts fail with no useful error in Event Viewer. The user gets a temporary profile and has no idea anything is wrong until their settings vanish.

Root Cause

The storage account's Kerberos ticket isn't being issued because the session host VMs were joined to Entra ID only — not hybrid-joined. Azure Files Kerberos auth requires the computer object to exist in on-prem AD.

# Check the join state on a session host dsregcmd /status | Select-String "AzureAdJoined|DomainJoined" # Both must be YES for FSLogix + Azure Files Kerberos auth AzureAdJoined : YES DomainJoined : YES # if NO — that's your problem

If DomainJoined : NO — your VMs aren't hybrid-joined. You need to re-image with the correct join type. This cannot be fixed post-deployment without re-imaging every VM.

Problem 2 — GPO Applying But Not Working

GPRESULT shows the FSLogix policy applied. The registry keys are there. But settings aren't taking effect and users still get temporary profiles.

Root Cause

A conflicting Intune MDM policy was winning over GPO. When MDM and GPO define the same setting, MDM wins. The FSLogix VHD location set via GPO was being overwritten by an Intune config profile pointing to a different path.

// lesson
Always run MDMDiagReport alongside GPRESULT on AVD session hosts. Intune doesn't announce conflicts.

Problem 3 — Azure Files Access Denied

Users authenticate successfully. But they get "Access Denied" when FSLogix tries to create the profile container.

Root Cause

Azure Files requires both RBAC (data plane access) AND NTFS permissions. RBAC alone isn't enough.

# Step 1: RBAC on the storage account # Storage File Data SMB Share Contributor → regular users # Storage File Data SMB Share Elevated Contributor → admins # Step 2: NTFS permissions — from a domain-joined host icacls \\storageaccount.file.core.windows.net\share /grant "domain\users:(M)" # ⚠ Cannot set NTFS perms from an Entra-only joined machine

Summary

IssueRoot CauseFix
Silent mount failureVMs not hybrid-joinedRe-image with hybrid join
GPO not taking effectIntune MDM conflictExclude VMs from Intune profile
Access Denied on shareMissing NTFS permissionsSet perms from domain-joined host

Lessons Learned

Plan the identity join type before deploying session hosts. Hybrid join cannot be retrofitted without re-imaging every VM.

Run MDMDiagReport on every session host as part of your checklist. When Intune and GPO clash, MDM wins silently.

Azure Files permissions are a two-layer model: RBAC for authentication, NTFS for authorisation. Set NTFS from a domain-joined machine — it won't work from an Entra-only host.

Azure Virtual Desktop
End-to-End Hybrid Deployment

avd azure fslogix identity networking
Problem

I needed hands-on experience with a full AVD deployment that mirrors production — not just spinning up a host pool, but the full stack: hybrid identity, FSLogix profiles, GPO, Conditional Access, and networking. AVD documentation covers each piece in isolation. This lab documents the full end-to-end.

Architecture
┌─────────────────────────────────────────────────────────┐ │ Azure Tenant │ │ │ │ ┌──────────────┐ ┌──────────────────────────┐ │ │ │ Entra ID │◄────►│ Azure AD Connect Sync │ │ │ └──────┬───────┘ └────────────┬─────────────┘ │ │ │ │ │ │ ┌──────▼───────────────────────────▼──────────────┐ │ │ │ AVD Host Pool (Pooled) │ │ │ │ ┌─────────────────┐ ┌──────────────────┐ │ │ │ │ │ Session Host 1 │ │ Session Host 2 │ │ │ │ │ │ (Hybrid Joined)│ │ (Hybrid Joined) │ │ │ │ │ └────────┬────────┘ └────────┬──────────┘ │ │ │ └────────────┼────────────────────┼───────────────┘ │ │ └──────────┬─────────┘ │ │ ┌────────────────────────▼──────────────────────────┐ │ │ │ Azure Files (Premium · LRS) │ │ │ │ FSLogix Profile Containers (VHDx) │ │ │ └───────────────────────────────────────────────────┘ │ │ ┌────────────────────────────────────────────────────┐ │ │ │ Conditional Access (AVD + MFA enforced) │ │ │ └────────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────┘ │ Azure AD Connect Sync ┌──────▼──────────┐ │ On-Premises AD │ (GPO source for FSLogix settings) └─────────────────┘
Steps
01.Create Resource Group, VNet (10.0.0.0/24) with AVD-Hosts-Subnet and AD-Subnet. Configure DNS to point to on-prem DC IP.
02.Deploy on-prem AD domain controller VM. Configure AD DS, DNS, create OU structure for AVD computer objects.
03.Configure Azure AD Connect on the DC VM. Enable Password Hash Sync. Verify user sync to Entra ID.
04.Create AVD Host Pool (pooled, BreadthFirst), Application Group, and Workspace. Assign users to Application Group.
05.Deploy session host VMs using the AVD gallery image (Windows 11 multi-session + M365). Configure as hybrid-joined during provisioning.
06.Create Azure Files Premium storage account. Enable Kerberos auth for AD DS. Assign Storage File Data SMB Share Contributor RBAC to Users group.
07.Set NTFS permissions on the share from a domain-joined host using icacls. Users: Modify. Admins: Full Control. SYSTEM: Full Control.
08.Create GPO targeting the AVD computer OU. Configure FSLogix: VHDLocations, DeleteLocalProfileWhenVHDShouldApply, FlipFlopProfileDirectoryName.
09.Create Conditional Access policy: require MFA for AVD app. Set Report-Only first, test thoroughly, then enforce.
10.Test: sign in as test user, verify FSLogix mounts in Event Viewer > FSLogix logs, confirm VHDx created in Azure Files share.
Issues Faced
IssueRoot CauseFix
FSLogix silent mount failureSession hosts Entra-only joined, not hybridRe-imaged with hybrid join at provisioning
GPO FSLogix settings ignoredIntune MDM policy conflictingExcluded session hosts from Intune device group
Azure Files Access DeniedRBAC set but NTFS permissions missingSet NTFS perms from domain-joined host
CA policy blocked all usersBreak-glass account not excludedAlways exclude break-glass. Always.
Lessons Learned
Plan the identity join type before deploying session hosts. Hybrid join cannot be retrofitted — re-imaging every VM is the only fix.
Azure Files permissions are a two-layer model: RBAC (authentication) + NTFS (authorisation). NTFS must be set from a domain-joined machine.
Always run CA policies in Report-Only mode for at least a week. Complete lockout — not degraded experience — is the failure mode.
Run MDMDiagReport alongside GPRESULT. When Intune and GPO collide on the same setting, MDM wins silently.
Cost for this lab: ~$1.80 AUD for 6 hours. Deallocate all VMs between sessions. Only the storage account persists (<$2/month).
